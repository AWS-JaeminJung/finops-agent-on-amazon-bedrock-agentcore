# Part 1. 개념 소개 (50분)

## 1-1. FinOps 개요 (10분)

### Cloud FinOps란?

FinOps는 **Financial Operations**의 약자로, 클라우드 비용에 대한 재무적 책임을 조직 전체가 공유하는 운영 모델입니다.

### FinOps의 3단계 사이클

```
Inform (가시화) → Optimize (최적화) → Operate (운영)
```

- **Inform**: 비용을 누가, 어디에, 얼마나 쓰고 있는지 가시화
- **Optimize**: 사용량 적정화, 요금 최적화, 아키텍처 개선
- **Operate**: 정책 수립, 자동화, 지속적 거버넌스

### 왜 AI Agent가 필요한가?

전통적인 FinOps 도구의 한계:

| 기존 방식 | AI Agent 방식 |
|-----------|--------------|
| 대시보드를 사람이 확인 | Agent가 능동적으로 이상 탐지 |
| 리포트를 수동 생성 | 자연어로 질의하면 즉시 분석 |
| 최적화를 개별적으로 적용 | 종합적 분석 후 우선순위 추천 |
| 정기적 점검 | 실시간 모니터링 및 알림 |

FinOps Agent가 다룰 수 있는 시나리오 예시:

- "지난 달 대비 비용이 가장 많이 증가한 서비스는?"
- "현재 사용률이 낮은 EC2 인스턴스를 찾아줘"
- "이번 분기 비용 예측을 해줘"
- "Savings Plans 구매를 추천해줘"

## 1-2. Kiro 소개 (20분)

### Kiro란?

Kiro는 AWS가 만든 **AI 기반 Agentic IDE**입니다. 단순한 코드 자동 완성을 넘어, 요구사항 정의부터 설계, 구현까지 소프트웨어 개발 전체 라이프사이클을 AI와 함께 수행합니다.

### Spec-Driven Development

Kiro의 핵심 차별점은 **Spec 기반 개발 워크플로우**입니다.

```
Requirements (요구사항) → Design (설계) → Tasks (구현 태스크)
```

#### 1) Requirements (요구사항)

자연어로 만들고 싶은 것을 설명하면, Kiro가 구조화된 요구사항 문서를 생성합니다.

- 사용자 스토리 형태로 정리
- GIVEN-WHEN-THEN 기반의 수락 기준(Acceptance Criteria) 자동 생성

#### 2) Design (설계)

요구사항으로부터 기술적 설계 문서를 생성합니다.

- 데이터 모델, API 스펙
- 기술 스택 및 아키텍처 결정
- 다이어그램 (Mermaid 형식)

#### 3) Tasks (구현 태스크)

설계를 기반으로 구체적인 구현 태스크를 생성합니다.

- 체크리스트 형태의 단계별 태스크
- 각 태스크를 클릭하면 Kiro가 자동으로 코드 구현
- 구현 후 자동으로 태스크 완료 체크

### Kiro vs 기존 AI 코딩 도구

| 비교 항목 | 일반 AI 코딩 도구 | Kiro |
|-----------|------------------|------|
| 작업 단위 | 프롬프트 단위 (vibe coding) | Spec 기반 (체계적) |
| 컨텍스트 유지 | 대화 세션 내 | Spec 문서에 영구 보존 |
| 결과물 | 코드 조각 | 요구사항 + 설계 + 코드 전체 |
| 협업 | 어려움 | Spec 문서를 통한 팀 공유 가능 |
| 예측 가능성 | 프롬프트에 따라 다름 | Spec이 가드레일 역할 |

### Steering Rules & Hooks

- **Steering Rules**: 프로젝트 전반에 적용할 코딩 컨벤션, 아키텍처 규칙을 설정
- **Hooks**: 파일 저장, 커밋 등 이벤트 발생 시 자동으로 실행할 동작 정의

## 1-3. Amazon Bedrock AgentCore 소개 (20분)

### AgentCore란?

Amazon Bedrock AgentCore는 **AI Agent를 프로덕션 환경에 배포하고 운영하기 위한 관리형 인프라 서비스**입니다. 어떤 프레임워크로 만든 Agent든, AgentCore 위에서 실행할 수 있습니다.

### 왜 AgentCore가 필요한가?

AI Agent를 로컬에서 만드는 것과 프로덕션에 배포하는 것 사이에는 큰 갭이 있습니다:

```
로컬 개발              프로덕션 운영
─────────────────────────────────────────
단일 사용자      →    다수 동시 사용자
수동 실행        →    API 호출로 실행
로컬 환경        →    확장 가능한 인프라
직접 디버깅      →    로깅/모니터링/트레이싱
하드코딩 키      →    보안 자격 증명 관리
```

AgentCore가 이 갭을 메워줍니다.

### 핵심 구성 요소

#### 1) Agent Runtime

- Agent 코드를 컨테이너화하여 실행
- 자동 스케일링 (요청에 따라 확장/축소)
- 프레임워크 무관: LangChain, CrewAI, Strands, 커스텀 등 모두 지원

#### 2) Tool Management

- Agent가 사용할 도구(Tool)를 중앙에서 관리
- AWS 서비스 연동 도구 기본 제공
- 커스텀 도구 등록 가능
- MCP(Model Context Protocol) 서버 연동 지원

#### 3) Memory & Session

- 대화 히스토리 관리
- 세션 간 컨텍스트 유지
- 장기 메모리 저장

#### 4) Observability

- 실행 로그 자동 수집
- 각 단계별 트레이스 (어떤 Tool을 어떤 순서로 호출했는지)
- CloudWatch 연동

#### 5) Identity & Access

- Agent에 IAM 역할 부여
- 사용자 인증/인가 연동
- 최소 권한 원칙 적용
